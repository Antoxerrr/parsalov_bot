Ну короче это парадигма объектно-ориентированного программирования, где всё представляется в виде классов и объектов

Класс это как чертеж какого-то объекта

Типа

class Cat {
constructor(color) {
this.color= color;
}

logColor() {
console.log(this.color)
}

meow() {
console.log("meow")
}
}

Допустим вот класс кота, но сам класс это лишь чертеж для объекта

У класса есть конструктор, это функция которая принимает различные параметры класса, вот допустим конструктор кота принимает цвет

Потом ты допустим решил создать котов

const leksik = new Cat("gray");
const casper = new Cat("white");

leksik.logColor();

Выведет gray

casper.logColor();

Выведет white

Тут ты уже никак не сможешь делать console.log(Cat.color) внутри функции logColor, потому что Cat это класс и он получает цвет только когда ты создаешь из него объект
