Есть ещё например такое понятие как полиморфизм, это когда разные объекты имеют метод с одним и тем же названием, но делают разное

Например у тебя есть класс Cat и класс Dog, у обоих сделаешь метод talk, у кошки он будет мяукать а у собаки он будет гавкать

Это полезно когда у тебя разные объекты обрабатываются каким-то одним обработчиком, и нужно чтобы каждый объект обрабатывался по своему, но при этом чтобы обработчик не "задумывался", какой ему объект пришёл, а просто у всех объектов вызывал один и тот же метод

Например ты делаешь игру змейка, у тебя есть класс яблока, и есть класс змейки, и отдельная функция, которая рисует все объекты на сцене

У обоих классов делаешь метод draw, который рисует объект на экране

И сколько бы ты разных объектов не добавлял, всякие там камни, отравленные яблоки, яблоки-бустеры, у всех просто делаешь метод draw, и просто всю пачку закидываешь в функцию-рисовалку, она просто у всех будет взывать draw, и ей будет похуй какой это объект и кто это такой

class Person {
constructor(name, age) { // вызывается при new
this.name = name;
this.age = age;
}

sayHello() { // автоматически в прототипе
console.log(`Привет, я ${this.name}`);
}
}

class Employee extends Person {
constructor(name, age, job) {
super(name, age); // вызывает constructor базового класса
this.job = job;
}

work() {
console.log(`${this.name} работает как ${this.job}`);
}
}

const e = new Employee('Мария', 28, 'разработчик');
e.sayHello(); // унаследованный метод
e.work();
